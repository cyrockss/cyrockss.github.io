Linux(4)--磁盘与文件系统

发表于 2017-03-18 | 分类于 Linux

本文介绍文件系统，文件系统管理

基本目录结构

/bin，/sbin：二进制程序，操作系统自身运行就需要的程序
/usr/bin，/usr/sbin：基本系统工具
/usr/local/bin，/usr/local/sbin：第三方程序
PS：*/sbin是root存放管理程序的目录

/lib，/usr/lib，/usr/local/lib：库文件
/etc：配置文件
/tmp：临时文件
/var/log，/var/mail，/var/cache：日志，邮件，内存

/proc：内核映像

  • /proc/cpuinfo - CPU 的信息(型号, 家族, 缓存大小等)
  • /proc/meminfo - 物理内存、交换空间等的信息
  • /proc/mounts - 已加载的文件系统的列表
  • /proc/devices - 可用设备的列表
  • /proc/filesystems - 被支持的文件系统
  • /proc/modules - 已加载的模块
  • /proc/version - 内核版本
  • /proc/cmdline - 系统启动时输入的内核命令行参数

/sys：硬件信息
/boot：引导目录，内核
/dev：硬件设备

  • b：随机设备
  • c：线性设备
  • /dev/hd[a-t]：IDE设备
  • /dev/sd[a-z]：SCSI设备
  • /dev/fd[0-7]：标准软驱
  • /dev/md[0-31]：软raid设备
  • /dev/loop[0-7]：本地回环设备
  • /dev/ram[0-15]：内存
  • /dev/null：无限数据接收设备,相当于黑洞
  • /dev/zero：无限零资源
  • /dev/tty[0-63]：虚拟终端
  • /dev/ttyS[0-3]：串口
  • /dev/lp[0-3]：并口
  • /dev/console：控制台
  • /dev/fb[0-31]：framebuffer
  • /dev/cdrom => /dev/hdc
  • /dev/modem => /dev/ttyS[0-9]
  • /dev/pilot => /dev/ttyS[0-9]
  • /dev/random：随机数设备
  • /dev/urandom：随机数设备

磁盘

硬盘：随机存取设备
I/O：

  • I/O控制器
  • I/O适配器

Trcak：磁道
Sector：扇区
Cylinder：柱面
RPM：转速

硬件接口：

  • IDE，ATA：并行，133M/s
  • SCSI：Small Computer System Interface 并行，320M/s
  • SATA：串行，300M/s，600M/s，6Gb/s
  • SAS：SCSI串行版
  • USB：（1.0,2.0,3.0）

在Linux下/dev/显示：
IDE，ATA：/dev/hd[a-z]
SCSI，SAS，SATA，USB：/dev/sd[a-z]
第一个IDE控制接硬盘
第二个IDE控制接光驱

文件系统

文件系统：通常由内核提供
Windows：
NTFS，FAT32

Linux：
ext2，ext3，ext4，xfs，reiserfs，nfs，iso9600，jfs，brtfs

文件系统格式化

对分区格式化就是创建文件系统
低级格式化：创建磁盘
高级格式化：创建文件系统

磁盘划分

RAW：裸设备
磁盘划分是根据柱面来划分的，不同的柱面上的相同磁道一定是相同的分区

MBR：Master Boot Record 主引导记录
第零个扇区
512bytes：

  • 446bytes：bootloader
  • 64bytes：分区表，其中每16bytes标示1个分区，最多4个主分区，主分区可作扩展分区，扩展分区再次划分为逻辑分区
  • 2bytes：5A 可作为引导盘

Linux：
sda[1-]
sda1，sda2，sda3，sda4
sda5，第一个逻辑分区

fdisk：
查看磁盘信息

partx：
让内核重新读取磁盘分区表

mkfs：
mke2fs：
格式化分区
配置文件：/etc/mke2fs.conf

tune2fs
显示块大小
一个块只能属于一个文件

文件系统挂载

mount：
挂载
挂载之后，原有的数据会被隐藏，因此不能挂载到系统常用目录上

umount：
卸载
卸载之时，要确保没有进程在访问挂载的设备，否则无法完成卸载

VFS：Virtual File System
统一各文件系统的open()，read()，write()，close()等接口

FileSystem：
一个软件，管理储存空间的文件，只有内核有权限访问硬件存储空间的文件

fuser：
查看正在访问某挂载点的进程

/etc/fstab
用空格隔开的六个字段

  1. 挂载设备：设备文件，卷标，UUID
  2. 挂载点
  3. 文件系统类型
  4. 挂载选项，default表示默认，多个选项逗号隔开
  5. 转储频率（备份频率）
    0：从不备份
    1：每日备份
    2：每隔一天备份
  6. 自检秩序
    0：不检测
    1：第一个检测，一般只有根文件系统

Linux的交换分区：swap

必须是独立分区（独立文件系统）

交换分区大小：
科学计算类服务器：交换内存尽可能大，比如物理内存的2倍
应用程序类服务器：交换内存尽可能小，比如只给1G

free：
查看内存
shared：共享内存
buffer：缓冲
cached：缓存

mkswap：
创建交换分区

swapon：
启用交换分区
swapoff
停用交换分区

文件系统格式

索引：存放文件属性，包括大小、属主属组权限、时间戳、对应的磁盘快指针，不存文件名
称：index node = inode

目录：文件的路径映射
inode 文件名 文件类型
一个目录本身只存其下的一级文件

inode bitmap：inode位图
block bitmap：块位图

block group：块组
100w = 5w * 20组

文件的复制、移动、删除

删除文件并不会清楚块中的数据，仅仅是将块放回空闲块中，将索引信息删除

链接文件
硬链接：指向同一inode的不同文件
软链接(符号链接)：ls -l显示文件类型为l的文件

硬链接：
ls -l第二段显示的数字指文件被硬链接的次数
硬链接不能跨分区指定
不能对目录创建硬链接，因为会导致循环引用
删除原文件路径，不影响链接文件的使用

符号链接：
不受分区限制
能对目录进行创建
删除原文件，连接文件将无法使用
链接文件与原文件指向不同inode，因此为不同文件
不识别文件内容，只识别文件名称
符号链接大小是指向文件路径字符串长度

压缩、解压缩、归档

gzip/tzip/xz
压缩比：xz > bzip2 > gzip
gzip：.gz gunzip = gzip -d zcat
bzip2：.bz2 bunzip2 = bzip2 -d bzcat
xz：.xz unxz = xz -d xzcat

压缩比：1-9 默认为6
压缩比越大，消耗的CPU越大

归档：archive

/dev/zero：0生成器
/dev/null：位桶

特殊权限
suid，sgid，sticky
