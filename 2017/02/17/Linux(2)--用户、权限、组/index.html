Linux(2)--用户、权限、组

发表于 2017-02-17 | 分类于 Linux

本文介绍Linux的用户、权限、组

用户基本概念

Linux是多任务多用户的操作系统

每个使用者都有用户标识和密码
Authentication(身份验证)
Authorization(授权)
Account/Audition(审计)

用户类别：

  • 管理员
  • 普通用户
    • 系统用户
    • 登录用户

命令提示符(prompt)：
root用户：#
普通用户：$

用户标识：UserID(UID)
16位的二进制数字0-65535

  • 管理员：0
  • 普通用户：1-65535
    • 系统用户：1-499(CentOS6) 1-999(CentOS7)
    • 登录用户：500-60000(CentOS6) 1000-60000(CentOS7)

名称解析：
Username <--> UID
根据名称解析库进行

组基本概念

组：
用户组，用户的容器

组标识和名称解析
Groupname <--> GID
解析库：/etc/group

组类别1：

  • 管理员组
  • 普通用户组
    • 系统用户组
    • 登录用户组

组类别2：

  • 用户的主组
  • 用户的附加组

族类别3：

  • 私有组：组名同用户名，且只包含一个用户
  • 公有组：组内包含了多个用户

认证管理

认证信息：
通过对比登录时提供的信息与事先存储的是否一致
password：
/etc/shadow
/etc/gshadow

密码使用的策略：

  1. 使用随机密码
  2. 最短长度不低于8位
  3. 大写字母、小写字母、数字标点符号组合
  4. 定期更换

加密算法：
plaintext –> ciphertext

  • 对称加密：加密和解密使用同一个密码
  • 非对称加密：加密和解密使用一对(两个)密钥
    密钥对：
    • 公钥：public key
    • 私钥：private key
  • 单向加密：只能加密，不能解密，提取数据特征码比较
    • 定长输出：与文件大小无关
    • 雪崩效应：一个微小的差别加密后结果差距很大
  • 算法：
    • md5：message digest 128位
    • sha：secure hash algorithm 160位
      sha224、sha256、sha384、sha512等

用户信息库：/etc/passwd
name：password：UID：GID：GECOS：directory：shell
name：用户名
password：可以是加密密码，也可以是占位符X
UID：用户ID
GID：用户所属的主组ID
GECOS：注释信息
directory：用户家目录
shell：用户的默认shell

影子化了的密码文件：/etc/shadow
用户名：加密的密码：最近一次修改时间：最短使用期限：最长使用期限：警告时间：过期期限：保留字段

组的信息库：/etc/group
groupname：password：GID：userlist
userlist：该组的成员，以此组为附加组的用户列表

用户和组管理

用户命令

useradd：
创建新用户或更新默认用户信息

创建用户的默认配置文件为/etc/login.defs
修改结果存放在/etc/default/useradd

usermod：
修改用户属性

userdel：
删除用户(默认不删除家目录)

passwd：
修改用户密码

组命令

groupadd：
创建新组

groupmod：
修改组属性

groupdel：
删除组

gpasswd：
修改组密码

组密码文件：/etc/gshadow

newgroup：
临时切换到一个新组

补充

chage命令：
更改密码过期信息

id命令：
显示用户实际和有效的ID

su命令(switch user)：
切换用户

管理员可无密码切换至其他任何用户

whoami命令：
查看当前用户

pwck命令：
密码检测

grpch命令：
组检测

权限管理

进程安全上下文：
进程对文件的访问权限应用模型：
进程的属主与文件的属主是否相同，如果相同则判断属组，都不是则使用other权限

权限的表示：
9位二进制表示，可写为3位八进制数字
左3位：user权限
中3位：group权限
右3位：other权限

rwx：
read：读权限
writer：写权限
excute：执行权限

文件：
r：可读取文件数据
w：可修改文件数据
e：可将此文件运行为进程

目录：
r：可使用ls命令获取其下所有文件列表
w：可修改此目录下的文件列表，即创建和删除文件
e：若可使用cd命令至此目录中，则可用ls -l获取文件详细信息

chmod：
权限管理命令，只能修改属于自己的文件
MODE表示法：

  1. 授权表示法：直接操作一类用户的所有权限位
  2. 授权表示法：直接操作一类用户的一个权限位
  3. 八进制授权法

从属关系管理命令：
只有管理员可以使用
chown：可修改用户和组
chgrp：只能修改组

PS：用户对目录有写权限，但是对其下的文件没有写权限，不能修改此文件，但可以删除此文件

umask：
文件的权限反向掩码，遮盖码
666-umask为文件默认权限
777-umask为目录默认权限
默认文件不能有执行权限，如果减得结果有执行权限，则需要将其加1

此设定仅对当前shell进程有效

install命令：
复制文件且设置文件属性，不能复制目录
功能：

  1. 单源复制
  2. 多源复制
  3. 创建目录

mktemp命令：
创建临时文件或目录
