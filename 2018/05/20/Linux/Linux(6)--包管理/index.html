Linux(6)--包管理

发表于 2018-05-20 | 分类于 Linux

本文介绍Linux包管理系统

rpm命令实现程序管理

安装：-ivh，–nodeps，==replacepkgs
卸载：-e，–nodeps
升级：-Uvh，-Fvh
查询：-q，-qa，-qf，-qi，-qd，-qc
检验：-V
导入GPG秘钥：–import，-K
数据库重建：–initdb，–rebuilddb

yum包管理工具

yum repository：yum repo
储存了众多的rpm包以及相关的元数据文件（放置在特定的目录下：repodata）

文件服务器：

ftp://
htttp://
nfs://
file://

yum命令用法

repolist [all|disable|enable]：显示仓库列表
deplist：显示包与包的依赖关系
list：显示程序包
install：安装程序包
update：升级程序包
remove：卸载包组
info：查看程序包信息
provides：查看指定特性(或文件)由哪个程序包提供的
clean：清除缓存
makecache：构建缓存
search：搜索程序包(模糊匹配)
reinstall：重新安装程序包
downgrade：降级程序包
history：查看yum事务历史
grouplist：显示包组
groupinstall：安装包组
groupupdate：升级包组
groupinfo：查看包组信息
groupremove：卸载包组

yum的命令行选项：

–nogpgcheck：禁止gpgcheck
-y：自动回复yes
-q：静默模式
–noplugins：禁止所有插件
–disablerepo=repoidglob：临时禁用此仓库
–enablerepo=repoidglob：临时启用此仓库

yum的repo配置文件

yum客户端：

配置文件：
/etc/yum.conf：为所有仓库提供公共配置
/etc/yum.repos.d/*.repo：yum仓库指向

仓库的定义：

[repositoryID]
name
baseurl=url://path/to/repository/
enabled={1|0}
gpgcheck={1|0}
gpgkey=URL
enablegroup={1|0}
cost

可使用的变量：

$releasever：当前OS的发行版主版本号
$arch：平台
$basearch：基础平台
$YUM0-$YUM9

源代码组织格式

多文件：源码中的代码之间很可能存在跨文件依赖关系

C，C++：make（configure –> Makefile.in –> makefile）
java：maven

源代码 –> 预处理 –> 编译 –> 汇编 –> 链接

编译安装三步骤：

  • ./configure：首先根据选项传递参数，指定启用特性、安装路径等，执行时会参考用户的指定以及Makefile.in生成makefile，然后检查依赖到的外部环境
  • make：根据makefile文件构建应用程序
  • make install

开发工具：
autoconf：生成configure脚本
automake：生成Makefile.in

建议：安装前查看INSTALL，README

开发程序源代码的获取：

  • 官方自建站点：
  • 代码托管：
    • SourceForge
    • Github.com
    • code.google.com

安装后配置

  1. 将二进制程序目录加到PATH环境变量中

  2. 导出库文件路径：编辑/etc/ld.so.conf.d/NAME.conf，添加新的库文件所在目录至此文件中，让系统重新生成缓存

  3. 导出头文件：基于链接的方式实现

  4. 导出帮助手册：编辑/etc/man.config文件，添加一个MANPATH
