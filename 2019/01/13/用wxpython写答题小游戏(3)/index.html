用wxpython写答题小游戏(3)

发表于 2019-01-13 | 分类于 Python

本文讲解怎么用wxpython写一个答题小游戏(3)–讲解三个答题按钮的处理，倒计时时间到的处理，答错题使用复活卡的处理

前言

上一篇，我们将给游戏界面增加了背景，实现了倒计时功能，构造了游戏结束界面，这一篇我们将补上游戏界面三个答题按钮的处理方法，倒计时时间到的处理方法，同时讲解答错题后自动使用复活卡弹窗的方法。

答案按钮点击处理

点击按钮后
第一步：停止倒计时
第二步：获取该按钮Label值（即回答的答案），并判断答案是否正确，如果正确，执行第三步，不正确执行第四步
第三步：回答正确，将该按钮背景颜色变为绿色，然后判断该题时候是第12题，如果是，游戏胜利；如果不是，继续回答下一题
第四步：回答错误，将该按钮背景颜色变为红色，并将正确答案的按钮背景色变为绿色。然后判断玩家是否还有复活卡，如果有，自动使用复活卡，然后继续游戏；如果没有，游戏失败
第五步：游戏胜利或游戏失败会跳转到游戏结束界面，若继续游戏更新游戏界面的题号，倒计时，复活卡次数，问题和三个答案

PS:
因为在Timer中调用的方法不能带参数，所以无法将玩家回答正确的题数传给游戏结束界面，因此游戏结束界面显示的回答正确数统一显示为题号减1，所以我们在回答完最后一道题也需要将题号加1
回答错误使用复活卡，题号不变
可以使用按钮的SetBackgroundColour(“”)方法将按钮背景色恢复(颜色字符串为空)
这里只写了第一个按钮的处理方法，其他两个按钮处理相同，这里不再赘述

游戏结束

PS:
我们在答完题会显示正确和错误答案1s，也就是1s后才执行更新游戏界面或者跳的游戏结束界面，更新游戏界面我们使用threading的Timer类，而跳转到游戏结束界面我们使用wx的Timer类
使用threading的Timer更新游戏界面后，倒计时的定时器会继续生效
前面提过跳转到游戏界面不能使用threading的Timer类，否则游戏结束界面会一闪而过
