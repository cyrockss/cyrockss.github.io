用wxpython写答题小游戏(1)

发表于 2019-01-01 | 分类于 Python

本文讲解怎么用wxpython写一个答题小游戏(1)，构造游戏开始界面，构造游戏界面前的准备

前言

公司最近有个答题赢积分的活动，奈何自己知识储备不够，老是打不过，就打算自己也写个答题的小游戏玩，顺便练习下新学的wxpython和多线程

游戏规则：

  1. 答对12题算赢
  2. 每人1张复活卡，回答错误可以复活1次，继续答题
  3. 每道题需在10s内回答，若没回答，视为回答错误
  4. 错误答案标记为红色，正确答案显示绿色，回答错误会显示正确答案

架构方式

我们采用传统的UI层，逻辑层，数据层三层架构

UI层：
写主函数入口和构造的UI

数据层：
写配置项和题库

逻辑层：
处理一些UI层不方便处理的代码，比如说获取一个问题，这个在UI层不方便处理，我们可以放在逻辑层处理

Frame

我们需要构造三个Frame：

第一个Frame是游戏准备界面，显示一个开始游戏按钮和一个设置按钮

第二个Frame是游戏界面，需要显示以下元素：

  • 玩家当前回答到第几题
  • 倒计时
  • 复活卡剩余次数
  • 问题
  • 三个回答按钮

第三个Frame是游戏结束界面，显示玩家答对了多少题，胜负判断

配置文件

在构造游戏界面之前，我们先考虑游戏的一些基础数据，我们可以将一些数据设置为可配置项来控制游戏难度。
我们新建一个py文件命名为know_conf.py，写入两个常量和两道例题，
我们的题目使用一个列表的数据结构，第一个字段是问题，第二三四个字段是三个可选答案，第五个字段是正确答案
题库是由题目构成的一个列表

获取一道题目

每次启动游戏，我们都希望从题库随机获取一道题，之后的每一题都应该是本次游戏未回答过的，我们应该将这部分放在逻辑层处理
新建一个py文件命名问know_logic.py
构造一个题库类，写初始化方法和获取一个题目的方法
